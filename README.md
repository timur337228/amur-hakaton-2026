# Budget Analytics

Система для загрузки бюджетных данных, поиска показателей на естественном языке и построения сводной аналитики по объектам, организациям, кодам и периодам.

Проект сделан в виде двух независимых слоёв:

- `FastAPI` отвечает только за API, импорт данных, аналитику, экспорт и speech-to-text
- `Django` отвечает за пользовательский сайт

## Что умеет проект

- **Автоматически подключать стартовый набор данных.** При запуске система может подхватить заранее подготовленные CSV из `project_file`, создать импортный пакет и сразу подготовить данные для аналитики. Это удобно для демо, продакшен-витрины и сценария, когда набор уже лежит на сервере.

- **Загружать пользовательские данные несколькими способами.** Проект принимает архивы `.zip`, `.rar`, `.7z`, папки с файлами и наборы отдельных CSV. Во фронтенде это работает через drag-and-drop или кнопки выбора архива/папки, а в API - через отдельные ручки импорта.

- **Сохранять структуру загруженного набора.** При загрузке папки фронтенд передаёт относительные пути файлов, поэтому система понимает, из какой части набора пришёл каждый CSV. Это помогает корректно разделять источники и не терять контекст исходных файлов.

- **Обрабатывать импорт в фоне.** После загрузки создаётся `batch`: пользователь не ждёт синхронной обработки всего архива, а интерфейс опрашивает статус импорта. Для каждого пакета видны состояние, количество файлов, число CSV, импортированные строки, нормализованные записи и ошибки.

- **Показывать статистику и превью данных.** После завершения импорта можно посмотреть диапазон дат, доступные показатели, источники данных, количество строк и первые записи набора. Это помогает быстро проверить, что данные распознаны и готовы к анализу.

- **Нормализовать бюджетные данные в единую модель.** Разные CSV приводятся к общей структуре бюджетного факта: источник, файл, бюджет, объект, организация, документ, дата, год, месяц, коды классификации, источник средств, показатель и значение.

- **Работать с несколькими группами источников.** Поддерживаются источники `РЧБ`, `Соглашения`, `Госзаказ` и `БУАУ`. По ним можно фильтровать данные, строить отдельные сводки и сравнивать показатели между источниками.

- **Считать ключевые бюджетные показатели.** В аналитике доступны лимиты, подтверждённые лимиты по БО, подтверждённые лимиты без БО, остаток лимитов, кассовые выплаты, суммы соглашений, суммы контрактов, платежи по контрактам и выплаты БУАУ.

- **Строить аналитику по обычному текстовому запросу.** Пользователь может написать вопрос естественным языком, например `Покажи лимиты по Благовещенску по месяцам`. LLM интерпретирует запрос и превращает его в параметры аналитики: показатели, период, фильтры и группировки.

- **Сохранять ручной контроль над фильтрами.** Текстовый запрос можно дополнять ручными полями: объект, организация, бюджет, даты, источник средств, КФСР, КЦСР, КВР, КВСР, КОСГУ/КЭСР, КОСГУ, код цели, номер и ID документа.

- **Объединять LLM-интерпретацию и ручные фильтры.** Если пользователь указал параметры руками, они имеют приоритет и аккуратно дополняют результат работы LLM. Это позволяет быстро формулировать запрос текстом, но точно фиксировать важные ограничения.

- **Работать даже без доступной LLM.** Если ключ, модель или внешний сервис временно недоступны, система не ломает основной сценарий: аналитика и Excel-экспорт продолжают работать по ручным фильтрам, а интерфейс показывает предупреждение.

- **Автоматически подстраховывать текстовые запросы.** В сервисе есть простая логика безопасного разбора: если в тексте найден конкретный год, он превращается в диапазон дат; если встречаются слова про лимиты, выплаты, контракты или соглашения, система может вывести подходящий показатель; если нет точного фильтра, используется текстовый поиск по данным.

- **Группировать результат разными способами.** Сводку можно строить по дню, месяцу, году, бюджету, объекту, организации, источнику, показателю, кодам классификации, коду цели и источнику средств.

- **Показывать итоговые суммы и процент исполнения.** API возвращает общую сводку по выбранным показателям, а также процент исполнения, когда в результате есть лимиты и кассовые выплаты.

- **Показывать таблицу результата.** Для выбранной группировки система возвращает строки с измерениями, показателем и суммой. Есть ограничение выдачи и смещение, чтобы можно было работать с большими наборами данных.

- **Строить графики для визуального анализа.** Интерфейс показывает динамику по месяцам, накопительный итог, структуру по показателям и сводку по годам. Это помогает не только получить число, но и увидеть изменение показателей во времени.

- **Давать готовые демо-запросы.** В интерфейсе есть быстрые примеры: лимиты по Благовещенску, кассовые выплаты по КФСР `0502`, контракты по источнику `gz`. Для них подготовлены стабильные ответы, чтобы демонстрация открывалась быстро.

- **Подсказывать доступные значения фильтров.** После импорта API возвращает списки объектов, организаций, бюджетов, кодов, источников средств, документов, показателей и групп источников. Эти значения используются в интерфейсе как подсказки.

- **Принимать голосовой запрос.** Пользователь может надиктовать вопрос с микрофона, после чего система распознаёт речь и подставляет текст в поле аналитического запроса.

- **Работать с Whisper двумя способами.** Поддерживается `Whisper API` через `302.ai` и локальный `faster-whisper`. Провайдер, модель, язык, таймаут, формат ответа, VAD-фильтр и устройство настраиваются через `config.yaml` и переменные окружения.

- **Возвращать подробный результат распознавания речи.** API отдаёт исходный текст, нормализованный текст, признак применения коррекции, язык, длительность аудио и слова с таймкодами, если они доступны.

- **Опционально улучшать распознанный текст через LLM.** Если включить `whisper.postprocess_with_llm`, система может поправлять типичные ошибки распознавания с учётом контекста набора данных, например названия городов, объектов или бюджетных терминов.

- **Экспортировать аналитику в Excel.** Кнопка `Скачать Excel` формирует не сырую CSV-выгрузку, а готовый аналитический `.xlsx`-отчёт по текущему запросу.

- **Формировать Excel с несколькими листами.** В файле есть `Параметры`, `Сводка по объектам`, `Итоги по объектам`, `Динамика` и `Детализация`. Поэтому результат можно сразу смотреть, пересылать или использовать как основу для отчёта.

- **Разделять публичную витрину и режим загрузки.** При `deploy: true` пользователь видит уже подключённый набор данных, но не может загружать свои архивы и папки. Интерфейс отключает drag-and-drop, а API блокирует прямые ручки загрузки.

- **Давать полноценный веб-интерфейс.** Django-сайт объединяет загрузку данных, статус импорта, сводку набора, текстовый и голосовой запрос, ручные фильтры, графики, таблицу результата, превью исходных данных и Excel-экспорт на одной странице.

- **Предоставлять API для интеграций.** FastAPI-слой отдаёт отдельные эндпоинты для импорта, статуса batch, списка файлов, ошибок, статистики, превью, доступных фильтров, аналитического запроса, интерпретации текста, подготовленных примеров, speech-to-text и XLSX-экспорта.

- **Хранить данные в PostgreSQL и управлять схемой через Alembic.** Импортированные факты, файлы, ошибки и служебные сущности живут в базе, а структура обновляется миграциями.

- **Запускаться локально и в контейнерах.** Проект можно поднять напрямую через Python или через `Docker Compose`. Для продакшена есть отдельный `docker-compose.prod.yml`.

- **Работать за reverse proxy с HTTPS.** В продакшен-сценарии `Caddy` принимает внешний трафик, проксирует запросы к Django и FastAPI и автоматически выпускает HTTPS-сертификаты при корректно настроенном домене.

- **Готовить данные для дообучения модели.** В репозитории есть SFT-датасет и скрипты для задачи `text query -> JSON filters`, чтобы улучшать качество преобразования пользовательских запросов в параметры аналитики.

- **Поддерживать проверку качества через тесты.** В проекте есть тесты для сайта, аналитики, импортных задач, роутеров импорта, Excel-экспорта, LLM-интерпретации и голосового ввода с моками внешних сервисов.

## Стек

- Backend API: FastAPI
- Веб-приложение: Django
- База данных: PostgreSQLPRO
- ORM и миграции: SQLAlchemy + Alembic
- Контейнеризация: Docker, Docker Compose
- Reverse proxy и HTTPS: Caddy
- Работа с речью: Whisper (API и локальный faster-whisper)
- LLM: YandexGPT (дообучение через SFT) или API 302.ai Deepseek-v4 fast
- Frontend: Django templates (SSR)
## Архитектура

- `core/api` — API на `FastAPI`
- `core/web` — сайт на `Django`
- `training/llm_sft` — датасет для дообучения модели
- `notebooks` — ноутбуки для экспериментов и обучения
- `scripts` — утилиты генерации датасета и запуска обучения
- `project_file` — стартовый набор CSV
- `storage` — импортированные данные и служебные файлы

## Быстрый старт

### 1. Установка зависимостей

```bash
python -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt
```

### 2. Настройка окружения

Создай `.env` на основе `.env.example`.

Минимально обычно нужны:

- `POSTGRES_DB`
- `POSTGRES_USER`
- `POSTGRES_PASSWORD`
- `LLM_API_KEY`
- `DJANGO_SECRET_KEY`

### 3. Запуск через Docker

Для локальной разработки:

```bash
docker compose up --build
```

После запуска будут доступны:

- API: `http://localhost:8000`
- сайт: `http://localhost:8001`

## Локальный запуск без Docker

### API

```bash
.venv/bin/python -m alembic upgrade head
.venv/bin/python -m uvicorn core.api.app.main:app --reload --port 8000
```

### Сайт

```bash
.venv/bin/python manage.py runserver 8001
```

По умолчанию сайт обращается к API по `http://localhost:8000`.

## Импорт данных

Есть три основных сценария:

- автоимпорт стартового набора из `project_file`
- загрузка архива
- загрузка папки/набора файлов

После запуска импорта создаётся `batch`, который обрабатывается в фоне. Затем фронтенд опрашивает его статус и показывает результат.

Полезные эндпоинты:

- `POST /api/v1/imports/default`
- `POST /api/v1/imports/archive`
- `POST /api/v1/imports/files`
- `GET /api/v1/imports/{batch_id}`

## Аналитика

Основной сценарий:

1. данные импортированы
2. пользователь вводит текстовый запрос или заполняет фильтры
3. API интерпретирует запрос
4. строится сводка, таблица, графики и Excel

Примеры запросов:

- `Покажи лимиты по Благовещенску по месяцам`
- `Покажи кассовые выплаты по 0502`
- `Покажи сумму контрактов по источнику gz`

Для этих трёх демо-запросов в интерфейсе используются заранее подготовленные ответы, чтобы результат открывался быстрее и стабильнее.

## Голосовой ввод

Поддерживаются два режима:

- `Whisper API` через `302.ai`
- локальный `faster-whisper`

Настройка идёт через `config.yaml`:

- `whisper.provider: api | local`
- `whisper.api_model`
- `whisper.local_model`
- `whisper.language`
- `whisper.postprocess_with_llm`

Сейчас дополнительная постобработка распознанного текста через LLM по умолчанию выключена, чтобы не замедлять систему.

## Режим деплоя

В `config.yaml` есть флаг:

```yaml
deploy: true
```

Если он включён:

- пользователь не может загружать свои архивы и папки
- сайт работает как витрина на уже подключённом наборе данных
- drag-and-drop в интерфейсе отключён
- API-ручки прямой загрузки блокируются

## Excel-экспорт

Экспорт формирует не сырую выгрузку CSV, а аналитический отчёт.

В файле есть листы:

- `Параметры`
- `Сводка по объектам`
- `Итоги по объектам`
- `Динамика`
- `Детализация`

Это соответствует задаче: пользователь выбирает объект, показатели и период, а на выходе получает сводный файл с лимитами, обязательствами, кассовыми выплатами, соглашениями, контрактами и динамикой.
